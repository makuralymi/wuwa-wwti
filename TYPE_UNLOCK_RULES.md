# WWTI 图鉴类型达成规则

本文档基于当前 `index.html` 的实现整理，描述每个图鉴类型在测试中如何达成。

## 判定规则

每道题只会给一个标签加 1 分。答完 12 题后，程序会为每个图鉴类型计算一次类型分：

```text
类型分 = 该类型绑定的所有标签分数之和
```

只要类型分大于等于 2，就算命中。但如果同时命中多个类型，程序会按下面的优先级选择第一个：

```text
MXZ > KXZ > JLDS > MLXR > JSYG > CTRL > DLG > NGL > MZK > MMR > CLL > DEAD > WOC > SMD
```

因此，某些类型虽然在图鉴中存在，但会因为题库分布或优先级抢占而无法成为最终结果。

## 每个类型如何达成

| 类型 | 绑定标签 | 当前可达性 | 达成方式 |
| --- | --- | --- | --- |
| `MXZ` 米孝子 | `mx` | 可达 | `mx >= 2`。必须在 Q2 选择「竞品对比视频」，并在 Q8 的两个 `mx` 选项中任选一个。它是最高优先级，达成后会覆盖其他类型。 |
| `KXZ` 库孝子 | `kz` | 不可达 | 需要 `kz >= 2`，但 `kz` 只出现在 Q1 的两个选项里，同一题只能选一次，所以最高只能拿 1 分。 |
| `JLDS` 节奏大师 | `drama` / `rage` | 可达 | `drama + rage >= 2`。例如选择两个节奏/开骂相关选项。需要避免同时满足 `mx >= 2`，否则会先变成 `MXZ`。 |
| `MLXR` 麻辣仙人 | `waifu` | 可达 | `waifu >= 2`。在 Q3、Q6、Q9 中选择至少两个角色厨相关选项。需要避免先命中 `MXZ` 或 `JLDS`。 |
| `JSYG` 竞速野狗 | `speed` | 可达 | `speed >= 2`。在 Q3、Q6、Q9 中选择至少两个效率/强度相关选项。需要避免先命中 `MXZ`、`JLDS`、`MLXR`。 |
| `CTRL` 控制狂玩家 | `logic` / `guide` / `speed` | 可达 | `logic + guide + speed >= 2`。因为它排在 `DLG` 和 `MZK` 前面，所以 `logic >= 2`、`guide >= 2`、或 `logic + guide = 2` 都会优先进入 `CTRL`。如果 `speed >= 2`，会先进入 `JSYG`。 |
| `DLG` 毒攻略 | `guide` | 不可达 | 需要 `guide >= 2`，但 `CTRL` 也吃 `guide`，并且优先级高于 `DLG`。所以一旦 `guide >= 2`，最终会先判成 `CTRL`。 |
| `NGL` 内鬼佬 | `leak` | 不可达 | 需要 `leak >= 2`，但 `leak` 只出现在 Q9「内鬼爆料准不准」，最高只能拿 1 分。 |
| `MZK` 米中客 | `logic` / `critic` | 可达 | `logic + critic >= 2`，但要避免 `CTRL` 抢占。比较稳的路线是 Q3 选择「看完专门找槽点」、Q11 选择「先看评论区有没有骂错」，也就是 `critic = 2`。 |
| `MMR` 萌萌人 | `story` / `waifu` | 可达 | `story + waifu >= 2`。如果 `waifu >= 2` 会先进入 `MLXR`，所以更稳的路线是 `story = 2`，或 `story = 1` 且 `waifu = 1`。 |
| `CLL` 吃流量 | `chaos` / `traffic` / `ego` | 可达 | `chaos + traffic + ego >= 2`。例如选择两个乐子/流量/刷存在感相关选项。需要避免先命中更高优先级类型。 |
| `DEAD` 已退坑但没完全退 | `dead` | 不可达 | 需要 `dead >= 2`，但 `dead` 只出现在 Q12「每天还刷消息」，最高只能拿 1 分。 |
| `WOC` 卧槽观察者 | `normal` / `group` | 可达 | `normal + group >= 2`。可走 Q4「分享体验」+ Q11「转发给朋友一起审判」，或 Q4「找同温层」+ Q11「转发给朋友一起审判」。需要避免先命中更高优先级类型。 |
| `SMD` 沉默的大多数 | `silent` | 可达 | 这是默认兜底结果。没有其他类型达到 2 分时会进入 `SMD`；如果 `silent >= 2` 且没有更高优先级类型命中，也会进入 `SMD`。 |

## 当前不可达类型汇总

当前逻辑中不可达的类型有 4 个：

```text
KXZ, DLG, NGL, DEAD
```

原因分别是：

- `KXZ`：`kz` 只在同一题 Q1 出现，最多拿 1 分。
- `DLG`：`guide >= 2` 时会被更高优先级的 `CTRL` 抢走。
- `NGL`：`leak` 只出现一次，最多拿 1 分。
- `DEAD`：`dead` 只出现一次，最多拿 1 分。

如果希望所有图鉴都能被测出来，需要调整题库标签分布、降低部分类型阈值，或调整优先级/判定条件。
